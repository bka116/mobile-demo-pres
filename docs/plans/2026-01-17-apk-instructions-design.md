# Страница инструкции по установке APK (АС "ПУСК")

## Контекст
- Нужна отдельная HTML-страница с инструкцией по установке APK для планшетов инспекторов.
- Визуальный стиль должен соответствовать `demo4.html` (шрифты, фон, карточки, цветовая схема).
- Ссылка на APK будет добавлена позже (Яндекс.Диск), поэтому нужна заглушка.
- Скриншоты будут подготовлены позже, пока нужны плейсхолдеры и запись в `SCREENSHOTS_LIST.md`.

## Цели
- Четкая пошаговая инструкция установки APK.
- Дублирование ссылки: кнопка + текстовая гиперссылка.
- Легкая замена скриншотов без правок верстки.

## Не цели
- Интерактивная логика или слайдовая навигация.
- Изменение существующих демо-страниц.

## Структура страницы
1. Хиро-блок: заголовок, краткое описание, CTA-кнопка, текстовая ссылка.
2. Секция "Пошаговая установка": Шаг 1 (разрешения), Шаг 2 (установка).
3. Секция предупреждений: Google Play Protect, что делать при блокировке.
4. Секция "Если файл не открывается" + рекомендация отключить разрешение после установки.
5. Секция "Скриншоты" с 3 плейсхолдерами.

## Контент
- Русский текст на основе предоставленных инструкций (PDF), адаптированный под АС "ПУСК".
- Упоминание, что названия пунктов в Android могут отличаться.
- Четкие краткие пункты действий.

## Скриншоты
- `apk_screen_settings.png` — экран разрешений установки.
- `apk_screen_downloads.png` — папка "Загрузки" с APK.
- `apk_screen_install.png` — экран установки APK.

## Реализация
- Новый файл `apk-instructions.html` в корне проекта.
- Подключение `assets/css/demo4.css`, Tailwind CDN и существующих шрифтов.
- Локальные CSS-оверрайды в `<style>` для скролла и масштаба текста.
- Обновление `SCREENSHOTS_LIST.md`.

## Проверка
- Открыть `apk-instructions.html` в браузере и проверить адаптивность.
- Убедиться, что кнопка/ссылка кликаются и плейсхолдеры отображаются.
